## ⭐ **Review do Projeto Poffo Snacks**

O **Poffo Snacks** é um projeto dividido em múltiplas interfaces independentes, cada uma com um propósito específico dentro do fluxo de pedidos de um sistema de lanches. O repositório contém quatro módulos principais:

### 🍔 **WebSite**  
A interface principal voltada ao cliente.  
É onde o usuário visualiza o cardápio, navega pelos produtos e interage com a experiência geral do site.  
O foco aqui é a apresentação e a navegação simples.

### 🔐 **Login**  
Responsável pela autenticação do usuário.  
Inclui telas de login e criação de conta, funcionando como porta de entrada para as áreas restritas do sistema.  
A estrutura sugere um fluxo simples e direto, sem dependências externas complexas.

### 🖥️ **Admin**  
O painel administrativo do sistema.  
É onde o administrador acompanha pedidos, gerencia o fluxo e controla o funcionamento interno.  
O layout é focado em funcionalidade, priorizando a visualização rápida das informações.

### 🏷️ **Totem**  
Uma interface separada, pensada para uso presencial — como um totem de autoatendimento.  
O objetivo é permitir que o cliente faça pedidos diretamente, sem interação humana, de forma rápida e intuitiva.

---

# 🧩 **Arquitetura Geral**

O projeto segue uma estrutura modular, onde cada parte funciona como um mini‑site independente.  
Isso facilita manutenção, deploy separado e evolução de cada módulo sem quebrar os outros.
